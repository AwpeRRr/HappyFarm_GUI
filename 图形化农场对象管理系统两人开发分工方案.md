# 图形化农场对象管理系统两人开发分工方案

## 0. 方案定位

本方案适用于 2 人小组完成本科 Java 课程上机大作业：图形化农场对象管理系统。

建议优先使用 Swing 实现 GUI，原因是课程 Java 环境通常自带 Swing，不需要额外安装 JavaFX、SWT 或第三方库，便于老师运行和验收。

系统目标：

- 通过 GUI 管理农场对象。
- 支持添加、查询、照料、删除、清空、保存、读取、排序显示。
- 程序启动时扫描存档目录，并根据存档数量决定自动读取或新建农场。
- 保存为本地 txt 文件，下次启动可以恢复任意一次保存的数据。
- 代码中明确体现多线程、GUI 事件处理、Java 流操作、常用容器类、String、Math、Object 等知识点。

## 1. 项目总体架构

建议采用分层结构，避免所有逻辑都写在 GUI 文件中。

### 1.1 模块划分

| 模块 | 作用 | 建议包名 | 主要负责人 |
|---|---|---|---|
| 模型层 | 定义农场对象、动物、植物、工具等数据结构 | `happyfarm.model` | 成员 A |
| 业务逻辑层 | 管理农场行、添加、查询、照料、删除、清空 | `happyfarm.core` | 成员 A |
| 文件存档层 | 扫描存档、保存 txt、读取 txt、解析对象 | `happyfarm.io` | 成员 A |
| GUI 层 | 主窗口、三栏布局、按钮、输入框、结果区、对象按钮 | `happyfarm.gui` | 成员 B |
| 工具类层 | 排序、字符串处理、输入校验、提示信息格式化 | `happyfarm.util` | 成员 A 负责核心，成员 B 按需调用 |
| 启动层 | 程序入口、启动时扫描存档、初始化 GUI | `happyfarm` | 成员 B |

### 1.2 推荐目录结构

```text
code/
  src/
    happyfarm/
      Main.java
      model/
        FarmObject.java
        Animal.java
        Plant.java
        ToolItem.java
      core/
        Farm.java
        FarmResult.java
      io/
        SaveManager.java
      gui/
        FarmGUI.java
      util/
        SortUtil.java
        FarmUtils.java
  saves/
    default.txt
```

说明：

- `code/src` 存放 Java 源码。
- `code/saves` 存放 txt 存档。
- 存档目录不存在时，由 `SaveManager` 自动创建。
- GUI 不直接操作文件，而是调用 `Farm` 和 `SaveManager`。

## 2. 类设计建议

### 2.1 核心类总览

| 类名 | 所属模块 | 职责 |
|---|---|---|
| `FarmObject` | 模型层 | 抽象父类，保存所有农场对象的公共字段和公共方法 |
| `Animal` | 模型层 | 动物类，继承 `FarmObject`，实现动物照料行为 |
| `Plant` | 模型层 | 植物类，继承 `FarmObject`，实现植物照料行为 |
| `ToolItem` | 模型层 | 工具类对象，继承 `FarmObject`，实现工具使用或维护行为 |

| `Main` | 启动层 | 程序入口，启动时扫描存档并打开 GUI |

### 2.2 `FarmObject` 抽象父类

建议字段：

```java
protected String type;
protected String name;
protected int priority;
protected String status;
protected int careCount;
```

建议方法：

```java
public abstract String care();
public String toSaveString();
public String getDisplayText();
public String getType();
public String getName();
public int getPriority();
public String getStatus();
public int getCareCount();
public void setStatus(String status);
public void setPriority(int priority);
@Override public String toString();
@Override public boolean equals(Object obj);
@Override public int hashCode();
```

设计要求：

- `care()` 由子类实现，体现多态。
- `toSaveString()` 将对象转换成固定格式文本，供 `SaveManager` 保存。
- `toString()` 用于右侧结果区输出。
- `equals()` 和 `hashCode()` 可以按 `type + name` 判断对象是否相同，体现 Object 类方法重写。

### 2.3 具体对象类

#### `Animal`

建议字段：

```java
private int mood;
```

建议 `care()` 行为：

- 照料后 `careCount + 1`。
- 使用 `Math.random()` 或 `Math.round()` 生成随机心情变化。
- 返回类似：`已喂养动物 小牛，心情 +3，当前状态：开心`。

#### `Plant`

建议字段：

```java
private int growth;
```

建议 `care()` 行为：

- 照料后随机增加成长值。
- 使用 `Math.min()` 限制成长值不超过 100。
- 返回类似：`已浇水植物 玉米，成长值提升到 45`。

#### `ToolItem`

建议字段：

```java
private int durability;
```

建议 `care()` 行为：

- 表示维护工具或使用工具。
- 使用 `Math.max()` 限制耐久度不低于 0。
- 返回类似：`已维护工具 水壶，耐久度恢复到 90`。

### 2.4 `Farm`

建议数据结构：

```java
private ArrayList<LinkedList<FarmObject>> rows;
```

选择理由：

- `ArrayList` 适合按行号快速访问。
- `LinkedList` 适合每行中间插入和删除对象。
- 每一行可以直接对应 GUI 中间区域的一行按钮。

建议方法：

```java
public Farm(int rowCount);
public int getRowCount();
public ArrayList<LinkedList<FarmObject>> getRows();
public FarmResult addObject(int rowIndex, int position, FarmObject object);
public ArrayList<FarmObject> queryByName(String name);

public FarmResult removeObject(int rowIndex, int position);
public FarmResult clearFarm();
public FarmObject getObjectAt(int rowIndex, int position);
public ArrayList<FarmObject> getAllObjects();
```

注意：

- 对外输入的行号可以让用户按 1 开始输入。
- 内部数组下标统一按 0 开始处理。
- `FarmUtils` 负责把用户输入行号转换成内部下标。

## 4. 成员 A 的任务

成员 A 主要负责系统的核心数据和文件读写。成员 B 只通过成员 A 提供的方法操作农场，不直接改动内部容器。

- 定义所有农场对象的公共字段。
- 实现抽象照料方法。
- 实现动物、植物、工具三类对象。
- 保证每个对象都能转换成可保存的文本。

要写的类：

- `FarmObject`
- `Animal`
- `Plant`
- `ToolItem`

要写的方法：

- `FarmObject.care()`
- `FarmObject.toSaveString()`
- `FarmObject.getDisplayText()`
- `FarmObject.toString()`
- `FarmObject.equals(Object obj)`
- `FarmObject.hashCode()`

- `Plant.care()`

最终交付物：

要写的方法：

```java
public Farm(int rowCount);
```

容器设计：

和成员 B 的对接点：

最终交付物：

- 清楚的输入边界：行号、位置越界时返回失败结果，不让程序崩溃。

### 4.3 文件存档层

要完成的功能：

- 程序启动和读取时扫描 `code/saves` 文件夹。
- 返回所有 `.txt` 存档文件。
- 保存当前农场到指定 txt。
- 从 txt 读取并恢复 `Farm` 对象。
- 根据对象类型重新创建 `Animal`、`Plant`、`ToolItem`。

要写的类：

- `SaveManager`

要写的方法：

```java
public ArrayList<Path> scanSaveFiles() throws IOException;
public void saveFarm(Farm farm, Path saveFile) throws IOException;
public Farm loadFarm(Path saveFile) throws IOException;
public Path createSavePath(String saveName);
private FarmObject parseObjectLine(String line);
```

建议存档格式：

```text
ROWS|3
ITEM|1|0|ANIMAL|小牛|2|开心|1
ITEM|1|1|PLANT|玉米|1|成长中|3
ITEM|2|0|TOOL|水壶|3|可用|0
```

字段含义：

| 字段 | 含义 |
|---|---|
| `ROWS` | 农场总行数 |
| `ITEM` | 对象记录标识 |
| 第 2 个字段 | 行号，建议保存为 1 开始 |
| 第 3 个字段 | 位置编号，建议保存为 0 开始或 1 开始，但必须统一 |
| 第 4 个字段 | 对象类型，如 `ANIMAL`、`PLANT`、`TOOL` |
| 第 5 个字段 | 对象名称 |
| 第 6 个字段 | 优先级 |
| 第 7 个字段 | 状态 |
| 第 8 个字段 | 照料次数 |

流操作要求：

- 扫描目录：`Files.exists()`、`Files.createDirectories()`、`Files.list()`。
- 保存文件：`FileWriter`、`BufferedWriter`、`PrintWriter`。
- 读取文件：`FileReader`、`BufferedReader`。
- 解析文本：`line.trim()`、`line.split("\\|")`。
- 文件名处理：`endsWith(".txt")`、`substring()`。

和成员 B 的对接点：

- GUI 点击保存时，成员 B 调用 `saveFarm(farm, path)`。
- GUI 点击读取时，成员 B 调用 `loadFarm(path)` 并替换当前 `Farm`。
- GUI 打开存档选择框前，成员 B 调用 `scanSaveFiles()`。

最终交付物：

- `SaveManager` 源码。
- 至少 2 个手写或程序生成的 txt 测试存档。
- 错误存档的处理策略：格式错误时抛出异常，由 GUI 显示错误信息。

## 8. 技术重点落地说明

### 8.1 多线程怎么用

适合放入子线程的任务：

- 启动时扫描存档目录。
- 读取存档。
- 保存存档。
- 未来如果对象很多，排序或批量刷新前的数据整理也可以放入子线程。

不建议放入子线程的任务：

- 直接修改 Swing 组件。
- 简单按钮状态切换。
- 少量文本输入校验。

推荐实现：

- 成员 B 在 `FarmGUI` 中使用 `SwingWorker`。
- `doInBackground()` 中调用 `SaveManager` 的文件操作方法。
- `done()` 中刷新 GUI 和输出结果。

关键点：

- Swing 的界面更新应该在事件派发线程中执行。
- 子线程中出现异常，不能让程序直接退出，要把错误信息显示到右侧结果区。

### 8.2 GUI 事件处理怎么写

需要体现的事件：

- `初始化` 按钮：读取行数输入框，创建 `Farm`。
- `添加对象` 按钮：读取类型、名称、优先级、行号、位置，创建对象并添加。
- `查询` 按钮：读取名称，调用查询方法。
- `照料对象` 按钮：读取选中位置或输入位置，调用照料方法。
- `删除对象` 按钮：读取行号和位置，调用删除方法。
- `清空农场` 按钮：调用清空方法并刷新。
- `保存游戏` 按钮：弹出覆盖或新建选择。
- `读取存档` 按钮：弹出已有存档列表。
- `对象按钮`：点击后更新当前选中行号和位置。
- `排序按钮`：调用排序工具并输出到右侧。

实现要求：

- 对象按钮事件在 `refreshFarmPanel()` 中动态创建。

### 8.3 Java 流操作怎么完成保存和读取

保存流程：

1. `SaveManager` 检查 `saves` 目录是否存在。
2. 不存在则创建目录。
3. 使用 `FileWriter` 或 `Files.newBufferedWriter()` 打开目标 txt。
4. 第一行写入 `ROWS|行数`。
5. 遍历 `ArrayList<LinkedList<FarmObject>> rows`。
6. 每个对象写一行 `ITEM|...`。
7. 使用 `BufferedWriter` 或 `PrintWriter` 关闭资源，推荐 try-with-resources。

读取流程：

1. 使用 `BufferedReader` 逐行读取。
2. `trim()` 去掉空格。
3. 空行跳过。
4. 遇到 `ROWS` 行创建 `Farm`。
5. 遇到 `ITEM` 行使用 `split("\\|")` 切分字段。
6. 根据类型字段 `toUpperCase()` 判断创建 `Animal`、`Plant` 或 `ToolItem`。
7. 调用 `farm.addObject(...)` 放回指定行和位置。
8. 读取完成后返回 `Farm`。

必须注意：

- 文件不存在、格式错误、数字转换失败都要抛出或处理异常。
- 读取旧文件时，如果类型不认识，应提示“未知对象类型”。
- 保存和读取都要使用 UTF-8，避免中文乱码。

### 8.4 容器类怎么选择

| 数据 | 推荐容器 | 理由 |
|---|---|---|
| 农场所有行 | `ArrayList<LinkedList<FarmObject>>` | 行号访问快，每行可以插入删除 |
| 单行对象 | `LinkedList<FarmObject>` | 适合指定位置插入、删除 |
| 存档列表 | `ArrayList<Path>` | 便于传给下拉框或选择框 |
| 查询结果 | `ArrayList<FarmObject>` | 按查找顺序展示 |
| 按名称辅助查询 | `HashMap<String, ArrayList<FarmObject>>` | 可选，用于扩展快速查询 |
| 按类别分组 | `TreeMap<String, ArrayList<FarmObject>>` | 类别自动排序 |
| 排序结果 | `ArrayList<FarmObject>` | 使用 `Collections.sort()` 和 `Comparator` |

排序示例方向：

```java
Collections.sort(list, Comparator.comparing(FarmObject::getType));
Collections.sort(list, Comparator.comparingInt(FarmObject::getPriority).reversed());
```

### 8.5 String、Math、Object 怎么体现

String 使用场景：

- `split("\\|")`：解析存档字段。
- `trim()`：处理用户输入和文件行。
- `toLowerCase()`：查询时实现大小写不敏感。
- `toUpperCase()`：统一对象类型。
- `substring()`：处理文件名或展示长名称摘要。
- `equals()`：判断类型、状态、按钮命令。
- `contains()`：实现模糊查询。

- `Math.min()`：限制成长值最大 100。

Object 使用场景：

- `toString()`：右侧结果区输出对象详情。
- `equals()`：判断两个农场对象是否相同。
- `hashCode()`：配合 `equals()`，未来可放入 `HashMap` 或 `HashSet`。

- 变量名能表达含义。
- 每个类职责清楚，不把所有代码写在一个类里。
- 老师提问时，两名成员都能说明自己负责的类、方法和知识点。

- `code/src` 下完整 Java 源码。

- 一份简短说明文档，说明如何编译和运行。
- 本分工方案。
- 如果课程要求展示运行效果，可以补充 GUI 截图。
